   Problem: Users lose all data if they clear browser storage or switch devices
   Solution: Implement cloud synchronization

   Implementation Options:
   a) Firebase/Supabase Integration
      - User authentication (Google, GitHub, Email)
      - Real-time sync across devices

2. SPACED REPETITION ALGORITHM (SM-2 or Anki-style)
   Problem: Current system doesn't optimize review scheduling
   Solution: Implement proven spaced repetition algorithms

   What to Add:
   - SM-2 Algorithm (SuperMemo 2)
     * Easiness Factor per card
     * Interval calculation based on performance
     * Reviews scheduled for optimal retention

   - Card States
     * New: Never seen before
     * Learning: Recently introduced
     * Young: Recently graduated from learning
     * Mature: Long intervals (21+ days)
     * Relearning: Failed cards

   - Review Queue Management
     * Due today vs overdue cards
     * Daily review limit (e.g., 100 new, 200 reviews)
     * Ordering: overdue > learning > new

   - Statistics
     * Retention rate per deck
     * Forecast: cards due next 7 days
     * "Ease hell" detection (cards with EF < 1.3)

   Implementation:
   ```typescript
   interface Card {
     id: string;
     // ... existing fields
     easinessFactor: number; // 1.3 to 2.5
     interval: number; // days
     repetitions: number;
     dueDate: number; // timestamp
     state: 'new' | 'learning' | 'young' | 'mature' | 'relearning';
   }

   function calculateNextReview(card: Card, quality: 1|2|3|4): Card {
     // SM-2 algorithm implementation
     if (quality < 3) {
       return { ...card, interval: 1, repetitions: 0, state: 'relearning' };
     }

     const newEF = Math.max(1.3, card.easinessFactor + (0.1 - (4 - quality) * (0.08 + (4 - quality) * 0.02)));
     const newInterval = quality === 3 ?
       card.interval * 1.2 :
       card.interval * newEF;

     return {
       ...card,
       easinessFactor: newEF,
       interval: Math.round(newInterval),
       repetitions: card.repetitions + 1,
       dueDate: Date.now() + (newInterval * 86400000),
       state: newInterval >= 21 ? 'mature' : 'young'
     };
   }
   ```

   Files to Modify:
   - src/types/index.ts (update FlashCard type)
   - src/hooks/useStudySession.ts (replace current queue logic)
   - src/services/scheduler.ts (new file for SM-2)
   - src/components/Statistics.tsx (add retention graphs)

   Estimated Effort: 1-2 weeks

   References:
   - SuperMemo 2 Algorithm: https://super-memory.com/english/ol/sm2.htm
   - Anki Manual: https://docs.ankiweb.net/studying.html

8. AUDIO SUPPORT

   Solution: Add audio to flashcards

   Features:

     * Google TTS API
     * Amazon Polly
     * Browser Web Speech API (free)
   - Auto-play audio on card flip
   - Speed control (0.5x to 2x)
   - Repeat audio button

   Implementation:
   - Add audioUrl field to QuestionItem
   - Use Web Audio API for playback
   - Store audio as base64 (or external URLs)
   - TTS: use window.speechSynthesis API

   ```typescript
   function speakText(text: string, lang: string = 'en-US') {
     const utterance = new SpeechSynthesisUtterance(text);
     utterance.lang = lang;
     utterance.rate = 1.0;
     speechSynthesis.speak(utterance);
   }
   ```

   Estimated Effort: 1 week

CONCLUSION
=================================================================

Senko-2 is already a solid foundation for a flashcard application.
The suggestions above would transform it into a comprehensive,
production-ready learning platform competitive with Anki and Quizlet.

Priority Matrix:
- Highest Impact + Lowest Effort: Cloud sync, spaced repetition
- High Impact + Medium Effort: Statistics, rich editor, audio
- Medium Impact + Low Effort: Tags, themes, accessibility
- Low Impact + High Effort: Collaborative features, mobile apps

Focus on Priority 1 items first to make the app viable for
serious learners. Then expand based on user feedback.

The current codebase is well-structured and maintainable, making
it relatively easy to add these features incrementally.
